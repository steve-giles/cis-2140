eCommerce website for CSI-2140 eCommerce Template

Starting week 7 you'll start building the font end framework for a ecommerce website

Week 7 - Event Handling

This week you'll start implementing an eCommerce website using the attached template (eCommerce.zip). The template has some basic HTML, but is missing functionality to process user events. Your task is to implement some of the missing functionality to process user events.

Detect mouse movement events - Highlight/unhighlight a product image in response to mouse movement. Detect mouse click events - User requested to add an item to their cart in response to a mouse click. Detect when a page event - Page is loaded in response to a page being loaded in the browser.

Week 8 - Objects

This week you'll continue to work on the eCommerce system. You'll enhance a custom JavaScript object (user) by increasing the number of properties and adding a new method for your object.

When the user browses to the login page, and does not enter a user id and password, and clicks "Login," alert the user that the input is required (25 pts). When the user browses to the registration page, enters a password and confirm password, and the passwords do not match, alert the user "Passwords much match" and have the function return false (25 pts). When the user browses to the registration page, enters valid data, and clicks "Register," then alert the user with their full name.

Week 9 - Document Object Model (DOM)

This week you'll continue to work on the eCommerce system by learning how to programatically interact with the HTML document.

When the user browses to the login page, enters a user id and password, and clicks login, on the top right corner of the screen display the text "Welcome [userid]". Be sure to replace [userid] with the user id entered by the user (25 pts). When the user browses to the shop page, and adds an item to their cart, disable the button that was just clicked (25 pts). When the user browses to the shop page, and adds an item to their cart, in the top left corner of the screen, notify the user that the product was added to their cart by displaying the text "[productId] was added to your cart. Replace [productId] of the id of myObject (25 pts). When the user browses to the shop page, and adds an item to their cart, in the top right corner of the screen, increment the number of products in the cart, and display the text as "Products: [product count]" (25 pts).

Week 10 - Windows Object

This week you'll continue to work on the eCommerce system by learning how to interact with the Windows Object.

When the user browses to the shop page, and adds an item to their cart, in the top left corner of the screen display a message that the product was added to their cart, and programmatically set the message to gradually disappear (fade out) after five seconds. Without any user action, after five seconds the message should gradually disappear (fade out). The fade out time should take two seconds (40 pts). When the user navigates to the Product Details page, and clicks the “Return to previous page,” then programmatically navigate to the users previous page (20 pts). When the user navigates to the Product Details page, and clicks the print icon, open the Print Dialog Box and display a print view of the page (20pts). When the user navigates to the Product Details page, display an alert which contains the current url. (20pts)

Week 11 - Arrays

This week you'll continue to work on the eCommerce system by implementing an array, while also making use of JavaScript iterators, tables, conditional statements, and the DOM.

When the user places their mouse pointer over the product shopping cart icon located on the top left corner of the screen, and pop-up dialog is displayed. All changes are located in this dialog. If the shopping cart is empty, display a message “There are no items in your cart.” You can check if items are in their cart by checking the size (length) of the array (shoppingCart) (25 pts). Iterate the shoppingCart array, and dynamically display a table which contains all items in their cart. At a minimum, you should display each product image, description, product quantity, and a button to remove the item. See function loadPageData as an example of how to dynamically create the HTML using JavaScript (65 pts). When the user request to remove an item from their cart, call function removeFromCart. You can accomplish this by attaching the click event to your button. See the function addEventListener as an example (10 pts).

Week 12 - Strings, Cookies, and Date objects

This week you'll will provide the ability for the ecommerce site to remember the previously logged on user by working with Strings, Cookies, and the Date object.

When the user logs on with the “remember me” check box checked, store the user id in one cookie, and the date/time when the user logged in, in a second cookie (30 pts). When the user logs on with the “remember me” check box” not checked, then delete the previous two cookies. Note: there is no need to check if the cookies exists, instead just create and save the two cookies with an expiration date/time in the past.(20 pts). When the user loads the login page, and the user previously logged in with the “remember me” check box checked, then get the user id from the previously saved cookie, populate the user id input element with the user id read from the cookie, and check the “remember me” check box (25 pts). When the user loads the home page, and the user previously logged in with the “remember me” check box checked, then get the user id from the previously saved cookie, get the date time the user logged in from the other previously saved cookie, and in the top right corner of the screen display a message "Welcome back [userid]. Previous login at [previousDateTimeLogin" (25 pts).

Week 13 - Forms

This assignment will involve working with three forms: User Registration, User Login, and Contact Us.

User Registration – Enhance this page (registration.html) to submit the user entered data to a server side page for processing (myAccount.php). When the page form is submitted, call the JavaScript function registerUser() to perform client side validation (15 pts). User Login – Enhance this page (login.html) to submit the user entered data to a server side page for processing (myAccount.php). When the page form is submitted, call the JavaScript function userLogin() to perform client side validation (15 pts). Contact Us – Create a new page with a form. When the user enters the required data and complete the form, submit the user entered data to a serer side page for processing (myAccount.php). When the page form is submitted, call the JavaScript function registerUser() to perform client side validation (70 pts).
